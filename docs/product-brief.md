# Product Brief - 집집

## Product Name

집집

Internal codename:

- Budongsan Radar

Feature language:

- 집픽
- 오늘의 집픽
- 집집 브리프

Tagline candidate:

> 오늘 부동산 이슈, 쉽게 집어드립니다.

## Problem

부동산 정보는 양이 부족한 것이 아니라 해석과 큐레이션이 부족하다.

사용자는 뉴스를 보고도 다음을 바로 알기 어렵다.

- 이 이슈가 진짜 중요한가?
- 어느 지역에 영향이 있나?
- 무주택자, 1주택자, 전세 세입자, 투자자 중 누구에게 영향이 큰가?
- 지금 행동해야 하는 이슈인가, 참고만 하면 되는 이슈인가?

## Product Thesis

부동산 플랫폼의 첫 진입점은 매물이 아니라 정보 해석이어도 된다.

매물 플랫폼은 네이버부동산, 직방, 호갱노노 같은 강자가 이미 강하다. 반면 GeekNews처럼 링크 기반으로 부동산 이슈를 모으고, 토론하고, AI가 실수요자 관점으로 해석해주는 수직 커뮤니티는 빈틈이 있다.

## Core Value

사용자가 매일 5분 안에 오늘의 부동산 이슈를 파악하고, 자신에게 의미 있는 이슈만 골라볼 수 있게 한다.

공개 브랜드는 `집집`으로 간다. 짧고 발음이 쉽고, 반복어라 기억과 밈화에 유리하다. `집픽`은 메인 브랜드가 아니라 오늘의 핵심 이슈를 집어주는 기능/섹션명으로 사용한다.

## Primary Users

### 1. 실수요자

- 첫 집 구매 또는 갈아타기 고민
- 대출, 청약, 전세, 정책 변화에 민감
- 전문 용어가 많은 기사와 리포트가 부담스러움

### 2. 초보 투자자

- 정책, 금리, 지역 수급, 재건축, 경매 정보를 따라가고 싶음
- 커뮤니티의 과장된 의견과 데이터 기반 이슈를 구분하고 싶음

### 3. 업계 관찰자

- 공인중개사, 분양/시행/프롭테크 종사자
- 여러 출처의 부동산 이슈를 빠르게 모니터링하고 싶음

## MVP Scope

### In

- 링크 수집
- 링크별 제목/요약/원문 URL 저장
- 카테고리 태그
- 지역 태그
- AI 요약
- 영향 대상 분류
- 추천 또는 북마크
- 데일리 브리핑 페이지

### Out

- 매물 등록
- 중개 기능
- 결제
- 실명 인증
- 복잡한 커뮤니티 랭킹
- 자동 크롤링 대규모 파이프라인
- 투자 조언처럼 보이는 확정적 문구

## Categories

- 매매시장

## Region Tags

MVP에서는 너무 세분화하지 않는다.

- 전국
- 서울
- 경기/인천
- 지방 광역시
- 지방 중소도시

동네 단위 태그는 사용량이 생긴 뒤 확장한다.

## AI Summary Format

각 링크는 다음 형식으로 요약한다.

```text
3줄 요약:
- ...
- ...
- ...

왜 중요한가:
...

영향 대상:
- 무주택자: ...
- 1주택자: ...
- 전세 세입자: ...
- 투자자: ...

체크포인트:
- ...
- ...
```

## Success Criteria

MVP 성공 기준:

- 링크 50개 이상 등록 가능
- 홈에서 오늘의 이슈 10개 확인 가능
- 각 링크 상세에서 AI 요약을 볼 수 있음
- 카테고리/지역 필터가 작동함
- 사용자가 "오늘 뭐 봐야 하는지" 5분 안에 감을 잡을 수 있음

## Key Risk

1. 법적/신뢰 리스크
   - 투자 조언처럼 보이는 표현을 피해야 한다.
   - AI 요약은 참고 정보로 명확히 표시한다.

2. 콘텐츠 품질 리스크
   - 초반에는 자동화보다 관리자 큐레이션을 섞어 품질을 잡아야 한다.

3. 너무 넓은 범위
   - MVP는 링크 큐레이션과 요약에 집중한다.
   - 매물, 지도, 실거래가 시각화는 후순위다.

## Recommended First Version

관리자가 링크를 등록하면 AI가 요약하고, 사용자는 홈/카테고리/상세 페이지에서 읽는 읽기 중심 MVP로 시작한다.
