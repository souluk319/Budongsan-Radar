# 집집 Marketing & Design Quality Gate

작성일: 2026-06-14

## 목적

집집은 "정리된 데이터 화면"이 아니라, 오늘 부동산판을 대신 읽고 우선순위를 찍어주는 모바일 브리핑 서비스다.

다음 구현부터는 기능이 돌아가는지만 보지 않는다. 공개 화면이 실제 서비스처럼 보이는지, 사용자가 다시 들어올 이유가 있는지까지 통과해야 한다.

## 병렬 리뷰 체계

- 마케팅/제품 포지셔닝 리뷰: 첫 5초 가치, 재방문 이유, 카피, 신뢰 근거 확인
- 웹디자인/프론트 UX 리뷰: 첫 viewport 편집감, 모바일 밀도, 타입/색/여백, 데모 앱 냄새 제거 확인
- 메인 구현 책임: Michael이 두 리뷰를 통합해 코드와 검증까지 책임진다

## 핵심 진단

현재 집집이 허접해 보일 수 있는 이유는 기능 부족보다 UI 문법과 운영 신뢰 신호 부족이다.

- `샘플`, `seed`, `example.com` 냄새가 남으면 실서비스가 아니라 데모로 보인다
- 중요도/점수/신호가 있어도 산정 근거와 업데이트 시각이 없으면 장식처럼 보인다
- 첫 화면이 컴포넌트 나열처럼 보이면 "오늘 뭘 보면 되는지"가 약해진다
- 사용자 상황별 영향은 좋지만, "그래서 오늘 내가 뭘 확인해야 하는지"까지 밀어붙여야 한다
- 재방문 루틴이 없으면 하루 보고 끝나는 정보판이 된다

## 첫 화면 가치제안 후보

아래 중 하나가 첫 화면에서 5초 안에 이해되어야 한다.

1. 오늘 부동산 뉴스 중 내 상황에 영향을 주는 이슈만 5분 안에 집어드립니다.
2. 정책·금리·청약·전세 변화를 무주택자, 1주택자, 세입자, 투자자 관점으로 번역합니다.
3. 뉴스 제목보다 먼저, 오늘 시장에서 실제로 봐야 할 신호와 체크포인트를 보여줍니다.

## 홈 첫 viewport 기준

390px 모바일 첫 화면 안에 아래가 보여야 한다.

- `집집`
- `오늘의 판세`
- 2줄 이하의 오늘 결론
- `강한 신호 · 92` 같은 보조 신호
- 무주택자/1주택자/세입자/투자자 상황별 영향 4개
- `오늘의 집픽` 제목 첫 줄

첫 화면에서 보이면 안 되는 것:

- 필터 묶음
- 긴 설명문
- 반복 카드 3개 이상
- 개발/샘플 표시
- 로그인 이메일
- 과한 메타 정보

## 디자인 규칙

### 레이아웃

- 첫 viewport는 `판세 결론 -> 상황별 영향 -> 오늘의 집픽 -> 지역 요약`이 한 덩어리처럼 보여야 한다
- 데스크톱은 `max-width 1120px`, 상단은 `1fr + 320px` 정도의 2컬럼
- 모바일은 단일 컬럼
- `FilterBar`는 첫 화면 밖으로 내린다

### 타입

- 모바일 H1: 27-30px, line-height 1.12, 2줄 이하
- 데스크톱 H1: 40-44px, line-height 1.08
- 섹션 제목: 18-22px
- 피드 제목: 16-17px
- 본문: 14-16px, line-height 1.55
- `font-black`은 H1, 오늘의 집픽 제목, 핵심 라벨에만 제한한다

### 색과 표면

- 기본 배경: `#F8FAFC` 또는 `#F7F8FA`
- 표면: `#FFFFFF`
- 본문 텍스트: `#111827` 또는 `#14110F`
- 보조 텍스트: `#5E6673`
- 라인: `#E5E7EB`
- 따뜻한 강조 배경 `#FFF8EC`는 오늘 한 줄 같은 브리프 강조에만 쓴다
- 신호색은 강함 `#E5484D`, 관찰 `#F59E0B`, 안정 `#10B981`, 정보 `#3182F6`처럼 기능적으로만 쓴다

### 여백

- 모바일 좌우: 16px
- 모바일 섹션 간격: 20-28px
- 카드 내부: 16px
- 데스크톱 섹션 간격: 32-44px
- 큰 카드 내부: 24px
- 칩/CTA 터치 타깃: 최소 40px

## 컴포넌트별 기준

### SiteHeader

- 모바일 nav는 2개 이하로 줄인다
- 로그인 이메일은 첫 화면에서 숨긴다
- 헤더 높이는 48px 내외로 가볍게 유지한다

### DailyBriefLead

- 긴 서비스 설명 대신 오늘 결론을 먼저 보여준다
- H1은 2줄 이하
- 키워드 칩은 3개 이하

### TodayPickCard

- 카드가 아니라 브리프 리드처럼 보여야 한다
- `강한 신호 · 92 -> verdict 한 줄 -> 제목 -> 왜 봐야 하나` 순서
- 모바일에서도 상황별 영향을 숨기지 않는다

### AudienceImpactChips

- 첫 viewport 안으로 올린다
- 390px에서는 가로 스크롤보다 2x2 칩이 더 낫다
- 각 칩은 상태와 한 줄 행동을 같이 보여준다

### RegionFlowSection

- 단순 텍스트 리스트 금지
- `서울 HOT`, `경기/인천 WATCH` 같은 미니 온도판 구조
- 이슈 수, 방향, 짧은 근거를 붙인다

### IssueBriefList / LinkCard

- 분류 태그보다 신호 라벨, 제목, 한 줄 해석, 출처가 먼저 보여야 한다
- 카드 반복보다 행 단위 signal tape가 더 적합하다

## 금지 카피

- `MVP`
- `DB 연결`
- `실제 저장 지원`
- `필터 결과`
- `Latest Links`
- `부동산 레이더`
- `Budongsan Radar`
- `RADAR`
- `레이더 점수`
- 투자 조언처럼 읽히는 `오른다`, `내린다`, `사라`, `팔아라`
- `AI가 정확히 예측`류 과장

`샘플 데이터` 고지는 법적/신뢰상 필요하면 상세/메타 영역에만 작게 둔다. 공개 홈 첫 viewport에는 노출하지 않는다.

## 재방문 구조

다음 제품 루프에서 최소 하나는 실제로 구현하거나 UI에 설계해야 한다.

- 매일 오전 브리프: 오늘 기억할 한 줄, 오늘의 집픽, 내 상황 영향
- 어제 대비 변화: 지역/정책/금리/전세 신호 변화
- 내 상황 고정: 무주택자/1주택자/세입자/투자자 선택 기반 홈 개인화
- 주간 부동산 판세 리포트
- 저장한 이슈 후속 알림

## 통과 기준

- 첫 5초 안에 집집이 뭘 해주는지 설명되는가?
- 모바일 첫 화면 안에 오늘 결론, 상황별 영향, 오늘의 집픽이 들어오는가?
- 모든 중요도/신호에 최소 근거가 붙어 있는가?
- 공개 홈 첫 viewport에 개발/샘플/내부 용어가 없는가?
- 원문 링크, 출처, 업데이트 시각이 있어 실서비스처럼 보이는가?
- 카드 반복보다 편집자의 우선순위가 먼저 보이는가?
- 사용자가 내일 다시 볼 이유를 하나 이상 발견하는가?
- 390px에서 가로 스크롤, 텍스트 겹침, 과밀한 메타 정보가 없는가?
