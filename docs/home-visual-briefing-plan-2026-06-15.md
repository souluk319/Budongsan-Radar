# 집집 홈 비주얼 브리핑 개편 플랜 - 2026-06-15

> 2026-06-15 업데이트: 이 문서의 `DailySignalMap`/`HomeQuestionFace`
> 1차 방향은 이후 `HomeSignalBoard`로 대체했다. 최신 홈 방향은 질문
> 카드 보드가 아니라 `오늘의 부동산 신호판`, `판세 요약`,
> `주의/확인/관망/기회`, `오늘 할 일`로 이어지는 시장 신호판이다.
> `DailySignalMap`, `집픽 핀`, `오늘의 시장지도`, `이슈 압력`,
> `오늘 사람들이 묻는 것`, `집집 답`, `오늘 켜짐`은 공개 홈 첫 화면
> 기준에서 제거 대상이다.

## 한 줄 결론

집집 홈의 고정 방향은 `텍스트 브리프`도 아니고 `관제 대시보드`도 아니다.

집집은 **오늘 부동산판을 한 장의 시각 브리프로 먼저 보여주고, 바로 내 상황별 판단으로 번역하는 서비스**여야 한다.

이 방향은 흔들지 않는다.

## 문제 재정의

현재 production 홈은 이전보다 훨씬 정리됐지만, 첫 화면에서 강한 시각 장면이 약해져 `잘 정리한 숙제장`처럼 보일 위험이 있다.

반대로 예전 `오늘의 시장지도`는 시각적 임팩트는 있었지만, `이슈 압력`, 추상 차트, 여러 패널이 경쟁하면서 관제 대시보드처럼 보였다.

따라서 다음 루프의 목표는 둘 중 하나를 고르는 게 아니라, **대시보드의 시각적 힘은 가져오고, 대시보드의 복잡함은 버리는 것**이다.

## 리서치에서 확정한 기준

### 1. Axios 계열 브리핑

Axios Smart Brevity 원칙은 핵심 요소를 한 화면에 맞추고, 굵은 글씨와 bullet로 시선을 잡는 쪽이다. 중요한 점은 텍스트를 줄이는 것만이 아니라, 독자가 읽기 전에 잡히는 `anchor point`를 만든다는 것이다.

집집 적용:

- 첫 화면은 긴 설명보다 `오늘의 장면 -> 오늘의 결론 -> 오늘의 집픽` 순서여야 한다.
- 한 화면 안에서 시각 앵커와 판단 문장이 같이 보여야 한다.
- 문단은 짧게, 제목은 결론형으로 간다.

### 2. Redfin/Zillow 계열 시장 데이터

Redfin은 가격, 거래량, 재고, 금리, 수요 같은 지표를 큰 숫자와 차트로 보여주고, 각 지표가 무엇을 뜻하는지 설명한다. Zillow Market Heat Index는 지역별 수요/공급 균형을 `market temperature`처럼 해석 가능한 상태로 번역한다.

집집 적용:

- 숫자를 크게 보여주는 것보다 `뜨거움 / 관찰 / 차분함`처럼 사용자 언어로 번역해야 한다.
- 지역 흐름은 텍스트 리스트가 아니라 작은 시장 온도판이어야 한다.
- 수치가 없거나 근거가 약한 시각화는 크게 만들지 않는다.

### 3. Bloomberg 계열 시장 시각화

Bloomberg 차트의 핵심은 예쁜 장식이 아니라, 여러 자산/지표를 빠르게 비교해 판단을 돕는 것이다.

집집 적용:

- 시각화는 장식이 아니라 `오늘 뭘 먼저 볼지`를 정하는 도구여야 한다.
- 차트가 많으면 안 된다. 첫 화면에는 하나의 대표 시각화만 둔다.
- 시각화는 항상 집픽, 지역, 사용자 상황 중 하나와 연결되어야 한다.

### 4. 국내 부동산 서비스

호갱노노, 아실, 부동산지인, 리치고의 공통점은 처음부터 지도, 가격, 시세, 시장강도, 실거래가, 입주물량 같은 시각 데이터를 전면에 둔다는 점이다. 사용자는 들어오자마자 "여기는 부동산 데이터를 보는 곳"이라고 이해한다.

집집 적용:

- 집집도 첫 화면에 부동산 서비스다운 시각성이 필요하다.
- 다만 매물 지도나 단지 검색과 정면 경쟁하면 안 된다.
- 집집의 시각화는 `어디가 비싸냐`가 아니라 `오늘 뉴스와 근거가 어느 지역/상황을 흔드냐`여야 한다.

## 확정 제품 방향

### 추천안: Daily Signal Brief

홈 첫 화면을 `Daily Signal Brief` 구조로 고정한다.

```text
[헤더]

6월 15일 월요일 브리프 · 실제 수집 8건 · 근거 3개

오늘은 대출과 전세 이슈부터 보세요
매수심리는 살아나지만, 정책 확정 전 베팅은 위험합니다.

[DailySignalMap]
서울        경기/인천
대출/정책    전세/청약
오늘의 집픽 핀
무주택자 확인 · 세입자 주의

[오늘의 집픽]
짧게 편집한 제목
왜 지금 봐야 하나
그래서 오늘 할 일

[내 상황별 영향 2x2]
무주택자 / 1주택자 / 전세 세입자 / 투자자
```

## 홈 정보 위계

### 모바일 390px 첫 viewport

반드시 보여야 할 것:

1. 브랜드 `집집`
2. 날짜/실제 수집/근거
3. 오늘 결론 H1
4. `DailySignalMap` 상단 60-70%
5. `오늘의 집픽` 라벨 또는 제목 첫 줄

가능하면 보여야 할 것:

6. 내 상황별 영향 2개 이상

첫 viewport에서 내려야 할 것:

- 필터바
- 긴 지역 흐름 설명
- 반복 이슈 리스트
- 전세 안전 체크 CTA
- 추상 점수/압력 차트

### 데스크톱

2컬럼 유지.

- 왼쪽: 결론, DailySignalMap, 오늘의 집픽, 이슈 리스트
- 오른쪽: 내 상황별 영향, 지역 온도판, 전세 안전 체크

데스크톱도 정보량을 늘리되, 첫 시선은 `DailySignalMap` 하나로 고정한다.

## 컴포넌트 설계

### 1. `DailySignalMap`

새 컴포넌트. 홈의 시각 주인공.

입력:

- `regionFlows`
- `strongestLink`
- `primaryAudience`
- `keywords`
- `liveLinkCount`
- `evidenceCount`

UI:

- 지도 흉내를 과하게 내지 않고, 지역 타일 기반으로 시작
- 서울 / 경기·인천 / 지방 광역 / 전국 변수 4영역
- 각 영역에 `HOT`, `WATCH`, `MIXED`, `COOL` 상태
- 오늘의 집픽이 걸린 지역에 pin 표시
- 아래에 "오늘 흔드는 이슈" 2개: 정책/대출, 전세/월세 등
- 선/곡선은 장식으로 최소만 사용

금지:

- 복잡한 실제 지도 SVG
- 4개 이상 차트
- 의미 불명 막대 그래프
- `이슈 압력` 같은 내부 지표명

### 2. `DailyBriefLead`

현재 유지하되 축소한다.

변경:

- 모바일 H1 27-30px로 낮춤
- H1 2줄 제한에 맞게 copy를 더 짧게 생성
- 키워드 칩은 3개 이하
- `오늘 먼저 볼 것` 보조 라인은 `DailySignalMap` 아래로 이동 가능

### 3. `TodayPickCard`

카드가 아니라 브리프 리드처럼 만든다.

순서:

1. `오늘의 집픽`
2. 편집 제목
3. 왜 지금
4. 그래서 오늘 할 일
5. 출처/읽는 시간/근거

변경:

- 원문 뉴스 제목을 그대로 크게 쓰지 않는다.
- `editorialTitle` 또는 짧은 display title 생성 로직을 추가한다.
- 모바일에서도 `왜 지금`과 `내 상황 영향`을 완전히 숨기지 않는다.

### 4. `AudienceImpactChips`

모바일에서 가로 스크롤을 없앤다.

변경:

- 390px 기준 2x2 카드
- 각 카드에 상태 + 행동 한 줄
- 선택된 카드 상세는 아래 한 줄만

예:

```text
무주택자
확인
대출 조건 먼저
```

### 5. `RegionFlowSection`

사이드/하단 보조 시각화로 유지한다.

변경:

- 텍스트 리스트에서 미니 온도판으로 변경
- 각 지역은 `지역명 / 상태 / 이슈수 / 한 줄 이유`
- 데스크톱 사이드에 적합하게 압축

## 카피 원칙

### 써야 하는 말

- 오늘 먼저 볼 것
- 오늘의 집픽
- 내 상황별 영향
- 지역 온도
- 근거 있는 브리프
- 실제 수집
- 왜 지금
- 오늘 할 일

### 피해야 하는 말

- 시장지도
- 이슈 압력
- 강한 신호
- 전체 이슈
- 점수/스코어 중심 표현
- 예측처럼 읽히는 `오른다`, `내린다`, `사라`, `팔아라`

단, `시장지도`라는 말은 완전 금지가 아니라 화면 내부 제목으로는 약하다. 내부 구현명도 `MarketMap`보다 `DailySignalMap`이 낫다.

## 구현 순서

### Phase 1. 홈 구조 고정

목표:

- 첫 화면에 시각 앵커 복원
- 숙제장 느낌 제거
- 대시보드 과밀 방지

작업:

1. `src/components/daily-signal-map.tsx` 추가
2. `src/app/page.tsx` 상단 순서를 `DailyBriefLead -> DailySignalMap -> TodayPickCard`로 재배치
3. 데스크톱에서는 `DailySignalMap + TodayPickCard`가 좌측 주인공이 되도록 정리
4. `AudienceImpactChips`를 2x2 카드형으로 변경
5. `RegionFlowSection`을 미니 온도판으로 변경

검증:

- 390px 첫 화면에 H1, DailySignalMap, 오늘의 집픽 시작부가 보여야 함
- 390px 가로 overflow 없어야 함
- console warn/error 없어야 함

### Phase 2. 편집 제목/행동 문장

목표:

- 뉴스 원문 제목 냄새 줄이기
- "그래서 나는 뭘 해야 하지?"를 홈에서 답하기

작업:

1. `home-signals.ts`에 `displayTitle`, `todayActionLine` 파생 로직 추가
2. `TodayPickCard`에 `왜 지금`, `오늘 할 일` 표시
3. `DailyBriefLead` headline을 2줄 이하 문장으로 제한

검증:

- 원문 제목이 너무 긴 경우에도 카드가 숙제장처럼 보이지 않아야 함
- `오늘 할 일`이 사용자 상황과 연결되어야 함

### Phase 3. 검증 계약 수정

목표:

- 기존 대시보드 계약도 아니고, 텍스트 브리프 계약도 아닌 새 계약으로 고정

작업:

1. `scripts/verify-mvp.mjs`에서 아래 문구 확인:
   - `근거 있는 브리프`
   - `오늘의 집픽`
   - `내 상황별 영향`
   - `지역 온도` 또는 `지역 흐름`
   - `오늘 할 일`
2. `오늘의 시장지도`, `이슈 압력` 강제 확인은 제거
3. production smoke에 모바일 overflow 확인 유지

검증:

- `npm run lint`
- `npm run build`
- `npm run verify:mvp`
- 390px/1280px Chrome screenshot

## 실패 기준

아래 중 하나면 실패다.

- 첫 화면이 다시 관제 대시보드처럼 보인다.
- 첫 화면이 텍스트 숙제장처럼 보인다.
- `오늘의 집픽`이 첫 화면에서 늦게 보인다.
- 지역/상황 시각화가 장식이고 판단에 도움을 못 준다.
- 실제 근거 수, 출처, 기준일이 사라진다.
- 모바일에서 가로 스크롤이나 잘림이 생긴다.

## 성공 기준

390px 모바일에서 5초 안에 사용자가 이렇게 이해해야 한다.

> 오늘 부동산 이슈 중 어디가 흔들리는지 보이고, 내 상황에서는 뭘 먼저 확인할지 알려주는 서비스네.

데스크톱에서는 이렇게 보여야 한다.

> 뉴스 링크 모음이 아니라, 부동산 이슈를 시장/지역/상황 기준으로 편집해주는 브리핑 툴이네.
