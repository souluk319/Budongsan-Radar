# 집집 시각화 와우포인트 + API 장보기

작성일: 2026-06-14

## 한 줄 결론

성욱의 지적이 맞다. 집집의 와우포인트는 큰 헤드라인이나 굵은 글씨가 아니라, 부동산 상황을 한눈에 이해시키는 시각화여야 한다.

지금부터 홈/Part2의 중심은 `오늘의 말`이 아니라 `오늘의 그림`이다.

## 방향 수정

### 틀린 해결

- 헤드라인 글씨 키우기
- 숫자 크게 박기
- 막대/게이지를 장식처럼 넣기
- 샘플 데이터를 그럴싸하게 포장하기
- 금융 대시보드처럼 어둡고 무겁게 만들기

### 맞는 해결

- 실거래, 전월세, 청약, 관리비, 지역 통계를 한 장면으로 보여주기
- 사용자가 `내 동네`, `내 전세`, `내가 보는 단지`를 바로 떠올리게 만들기
- 데이터 신뢰도와 최신성을 작게 표시하기
- 불확실한 건 불확실하다고 표시하기
- 지도/흐름/비교/분포/타임라인을 제품의 주인공으로 만들기

## 시각화 제품 원칙

1. 차트는 장식이 아니라 판단 도구여야 한다.
2. 지도와 비교표는 예쁜 것보다 바로 읽혀야 한다.
3. 첫 화면에는 텍스트 결론보다 시각 장면이 먼저 보여야 한다.
4. 숫자는 크게 외치는 게 아니라 맥락 안에서 보여야 한다.
5. 실제 데이터가 없으면 시각화를 크게 만들지 않는다.
6. 모든 시각화에는 출처, 기준일, 샘플/실제 여부를 작게 붙인다.
7. `오른다/내린다/사라/팔아라` 같은 예측형 표현은 금지한다.

## 2026-06-14 실서비스 레퍼런스 관찰

성욱 피드백 이후 실제 서비스 첫 화면을 다시 확인했다. 결론은 간단하다. 인기 부동산 서비스는 설명부터 하지 않는다. 들어가자마자 지도, 가격 말풍선, 랭킹, 차트, 필터가 먼저 보인다.

### 호갱노노

- 첫 인상: 지도 위에 아파트 가격 말풍선과 필터가 바로 보인다.
- 강점: 사용자가 앱 성격을 1초 안에 이해한다. `부동산 데이터를 보는 곳`이라는 장면이 먼저 들어온다.
- 집집 적용: 텍스트 브리핑 상단보다 지도형 시장 보드가 먼저 나와야 한다.

### 리치고

- 첫 인상: 지도 위 가격 마커, 좌측 랭킹 패널, 상단 필터, 우측 기능 탭이 동시에 보인다.
- 강점: 다소 복잡해도 `부동산 빅데이터 도구`라는 인상이 즉시 생긴다.
- 집집 적용: `지금 뜨는 이슈 TOP`, `지역 말풍선`, `이슈 압력`을 한 장면에 같이 배치한다.

### 아실/부동산지인

- 첫 인상: 실거래/전세/시장강도/입주물량 같은 지표를 지도와 차트로 밀어붙인다.
- 강점: 부동산에 관심 있는 사용자가 다시 들어올 이유가 있다. 숫자가 아니라 비교와 흐름을 보러 온다.
- 집집 적용: Part2는 전세 안전 그림, 단지 스파크라인, 지역 시장지도 순서로 강화한다.

### Redfin/Zillow/Realtor.com

- 첫 인상: 시장 열기, 수요/공급, 지역 랭킹, 데이터 다운로드처럼 `시장 데이터`를 전면에 둔다.
- 강점: 콘텐츠 사이트가 아니라 시장 데이터 허브처럼 보인다.
- 집집 적용: 뉴스 링크 나열이 아니라 `오늘 어디가 흔들리는지`를 시장지도 형태로 보여준다.

## 이번 홈 반영 기준

- 홈 첫 화면 h1은 유지하되, 화면 주인공은 문장이 아니라 시장지도다.
- `오늘의 시장지도`에 지도형 배경, 지역 말풍선, 신호 스파크라인, 이슈 압력 막대, 이슈 랭킹을 넣는다.
- `오늘의 집픽`은 별도 카드가 아니라 시장지도 옆에서 현재 판세를 만든 핵심 이슈로 붙인다.
- `내 상황별 영향`은 하단 설명 카드가 아니라 첫 화면 우측 패널에 붙여 사용자 입장을 바로 보여준다.
- 실제 데이터 수집 수와 공식 근거 수를 첫 화면 상단에 노출한다.

## 집집의 핵심 와우포인트 후보

### 1. 오늘의 시장지도

첫 화면 상단에 `서울/경기/인천/지방 광역시` 흐름을 작은 지도나 타일 히트맵으로 보여준다.

사용자가 느끼는 것:

- 오늘 어디가 뜨겁고 어디가 식었는지 바로 보인다.
- 뉴스 제목을 읽기 전에 시장 분위기를 감으로 잡는다.
- 집집이 단순 뉴스앱이 아니라 데이터 브리핑처럼 보인다.

필요 데이터:

- REB/R-ONE 지역별 매매가격지수, 전세가격지수, 거래량, 매수심리 계열
- 국토부 실거래가 API 지역별 최근 거래 표본
- 지역 코드 정규화

현재 상태:

- `REB_STATS_API_KEY` 있음
- `DATA_GO_KR_SERVICE_KEY` 있음
- REB endpoint 매핑은 아직 필요

추가로 있으면 좋은 API:

```bash
JUSO_CONFIRM_KEY=
VWORLD_API_KEY=
```

### 2. 내 동네 온도 스트립

사용자가 지역을 누르면 `최근 거래`, `전세`, `청약`, `뉴스 신호`를 4칸짜리 스트립으로 보여준다.

예시:

```text
마포구
거래 회복 | 전세 압박 | 청약 강함 | 정책 영향 큼
```

사용자가 느끼는 것:

- "이건 전국 뉴스가 아니라 내 생활권 이야기네"가 된다.
- 지역 페이지로 들어갈 이유가 생긴다.

필요 데이터:

- 실거래가/전월세 API
- REB 지역 통계
- 네이버 뉴스 검색 결과의 지역 태그
- 주소/법정동 코드 매핑

추가로 있으면 좋은 API:

```bash
KAKAO_REST_API_KEY=
JUSO_CONFIRM_KEY=
VWORLD_API_KEY=
SEOUL_OPEN_DATA_API_KEY=
GG_DATA_API_KEY=
```

### 3. 전세 안전 그림

전세 안전 체크를 단순 점수표가 아니라 `보증금 위치`를 보여주는 그림으로 만든다.

예시:

```text
최근 매매가 범위     ━━━━━━━━━━━━━
최근 전세가 범위       ━━━━━━━
내 보증금 위치             ●
주의 구간                       ███
```

사용자가 느끼는 것:

- 내가 넣으려는 보증금이 위험 구간에 가까운지 눈으로 본다.
- 부동산 플랫폼에서 가장 강한 재방문 도구가 된다.

필요 데이터:

- 아파트/오피스텔/연립다세대/단독다가구 매매 실거래가
- 아파트/오피스텔/연립다세대/단독다가구 전월세 실거래가
- 건축물대장: 사용승인일, 용도, 층, 면적, 건물 구조
- HUG 전세보증금반환보증 통계
- 법제처/정책 문서

현재 상태:

- `DATA_GO_KR_SERVICE_KEY` 있음
- 실거래가 일부는 호출 가능
- 건축물대장 활용신청/adapter 필요
- HUG는 open API/파일데이터 확인 필요

성욱이 추가로 해주면 좋은 것:

- 공공데이터포털에서 아래 API 활용신청
  - 국토교통부 건축HUB 건축물대장정보 서비스
  - 아파트 매매 실거래가
  - 아파트 전월세 실거래가
  - 오피스텔 매매/전월세 실거래가
  - 연립다세대 매매/전월세 실거래가
  - 단독다가구 매매/전월세 실거래가
  - HUG 전세보증금반환보증 관련 데이터

### 4. 단지 비교 스파크라인

단지 페이지에서 `실거래`, `전세`, `관리비`, `연식`, `세대수`를 작은 선/점/띠로 비교한다.

사용자가 느끼는 것:

- 이 단지가 비싼 이유, 싼 이유, 불편한 이유를 빠르게 본다.
- "집집 들어가면 단지 감이 잡힌다"가 된다.

필요 데이터:

- 공동주택 단지 목록
- 공동주택 기본 정보
- 공동주택 관리비
- 장기수선충당금
- 회계감사보고서
- 건축물대장
- 실거래가/전월세

현재 상태:

- 기존 `DATA_GO_KR_SERVICE_KEY`로 활용신청 가능한 항목이 많다.
- 별도 `KAPT_API_KEY`가 아니라 공공데이터포털 서비스키를 쓰는 경우가 많다.
- K-apt 직접 포털에서 별도 키를 발급받는 구조가 있으면 그때 `KAPT_API_KEY`를 추가한다.

성욱이 추가로 해주면 좋은 것:

- 공공데이터포털에서 아래 API 활용신청
  - 국토교통부 공동주택 단지 목록제공 서비스
  - 국토교통부 공동주택 기본 정보제공 서비스
  - 국토교통부 공동주택관리비 개별사용료 정보제공서비스
  - 국토교통부 공동주택관리비 장기수선충당금 정보서비스
  - 국토교통부 회계감사보고서 정보

### 5. 청약 경쟁률 그림

청약 이슈는 숫자 나열이 아니라 `분양가`, `경쟁률`, `지역 쏠림`을 한 장에 보여준다.

예시:

```text
서울 핵심입지    ██████████  높음
경기 인기지      ██████      중간
지방 일부        ██          낮음
```

사용자가 느끼는 것:

- 청약 뉴스가 광고가 아니라 시장 온도처럼 보인다.
- 무주택자에게 바로 꽂힌다.

필요 데이터:

- 청약홈 분양정보
- 청약 경쟁률/특별공급 신청현황
- 지역별 분양가/공급량

현재 상태:

- `DATA_GO_KR_SERVICE_KEY` 있음
- 활용신청과 adapter 필요

### 6. 월 부담 시뮬레이터

정책/금리 뉴스는 `내 월 상환액이 얼마나 흔들리는지`로 보여준다.

예시:

```text
대출 3억 기준
금리 0.25%p 변화 -> 월 부담 약 n만원 변화
```

사용자가 느끼는 것:

- 추상적인 금리 뉴스가 내 생활비로 바뀐다.
- 토스식 효용이 생긴다.

필요 데이터:

- ECOS 기준금리/시장금리
- 주택금융통계
- 금융상품/대출 비교 데이터는 약관 확인 후

현재 상태:

- `ECOS_API_KEY` 있음

추가 후보:

```bash
HOUSTAT_API_KEY=
FSS_FINLIFE_API_KEY=
```

주의:

- 실제 대출 상품 추천/중개처럼 보이면 규제 리스크가 커진다.
- 초기에는 교육용 계산/시나리오로 제한한다.

### 7. 생활권 레이어

지역/단지 페이지에 지하철, 학교, 상권, 인구 흐름을 작은 레이어로 붙인다.

사용자가 느끼는 것:

- "여기 왜 인기 있지?"가 보인다.
- 단지 페이지가 단순 뉴스보다 오래 남는 SEO 자산이 된다.

필요 데이터:

- 서울 열린데이터광장: 지하철, 학교, 상권, 생활인구
- 경기데이터드림: 경기도 생활권/교통/인구
- 공공데이터포털 생활 SOC/교육/교통 데이터

추가 후보:

```bash
SEOUL_OPEN_DATA_API_KEY=
GG_DATA_API_KEY=
```

## 성욱이 추가로 받으면 좋은 키

### 최우선

```bash
KAKAO_REST_API_KEY=
JUSO_CONFIRM_KEY=
VWORLD_API_KEY=
```

- `KAKAO_REST_API_KEY`: 주소/장소 검색 UX, 지역/단지 검색
- `JUSO_CONFIRM_KEY`: 공식 도로명주소 검색/정규화
- `VWORLD_API_KEY`: 공공 공간정보, 좌표/지도 레이어

### 다음

```bash
SEOUL_OPEN_DATA_API_KEY=
GG_DATA_API_KEY=
```

- 서울/경기 생활권 시각화용
- 지하철, 상권, 인구, 학교 같은 재방문형 데이터에 필요

### 선택

```bash
HOUSTAT_API_KEY=
HUG_DATA_API_KEY=
FSS_FINLIFE_API_KEY=
```

- 주택금융통계, 전세보증 통계, 금융상품 비교용
- 초기에는 없어도 된다.

## 기존 키로 먼저 해야 할 활용신청

이미 `DATA_GO_KR_SERVICE_KEY`가 있으므로 새 env보다 활용신청이 더 중요하다.

### 실거래/전월세

- 국토교통부 아파트 매매 실거래가 자료
- 국토교통부 아파트 전월세 실거래가 자료
- 국토교통부 오피스텔 매매 실거래가 자료
- 국토교통부 오피스텔 전월세 실거래가 자료
- 국토교통부 연립다세대 매매 실거래가 자료
- 국토교통부 연립다세대 전월세 실거래가 자료
- 국토교통부 단독다가구 매매 실거래가 자료
- 국토교통부 단독다가구 전월세 실거래가 자료

### 건물/단지

- 국토교통부 건축HUB 건축물대장정보 서비스
- 국토교통부 공동주택 단지 목록제공 서비스
- 국토교통부 공동주택 기본 정보제공 서비스

## 내가 할 수 있는 것

### 바로 가능

- 위 키 이름을 `.env.example`에 반영
- `data adapters`를 원천별로 분리
- `EvidenceObservation`을 시각화 가능한 형태로 확장
- 지역/단지/전세 체크 화면에 실제 차트 컴포넌트 추가
- 홈 첫 화면을 `시장지도 + 오늘의 집픽` 구조로 재설계
- 샘플/실제/불확실성을 UI에 명확히 표시
- API health check에 추가 키 상태 표시

### 키가 들어오면 가능

- 주소 검색/정규화

## 구현 우선순위

### Phase A: 홈 와우포인트

- 홈 상단을 `오늘의 시장지도`로 바꾼다.
- 텍스트 헤드라인보다 지역별 흐름 그림이 먼저 보이게 한다.
- 실제 수집/공식 근거 개수를 작게 표시한다.

### Phase B: 전세 안전 시각화

- `/tools/jeonse-check`를 입력형으로 바꾼다.
- 보증금 위치, 최근 전세/매매 범위, 건축물대장 리스크를 그림으로 보여준다.

### Phase C: 단지 브리프

- `/complexes/[id]`를 단지 비교 카드로 바꾼다.
- 실거래/전세/관리비/연식/세대수 스파크라인을 붙인다.

### Phase D: 지역 페이지

- `/regions/[id]`에 지역 온도, 거래 흐름, 전세 흐름, 청약 흐름을 한 화면에 보여준다.
