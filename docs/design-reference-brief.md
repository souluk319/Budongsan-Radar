# Design Reference Brief - 집집

집집 홈 리디자인을 위한 레퍼런스 브리프.

목표는 "예쁜 SaaS 대시보드"가 아니라, 매일 들어와서 오늘 부동산 이슈를 빠르게 집어보는 제품 화면을 만드는 것이다.

## Reference 1. GeekNews

URL: https://news.hada.io/

### 가져올 것

- 링크 중심 피드의 밀도
- 제목, 출처, 요약, 점수, 댓글 같은 정보 우선순위
- 불필요한 장식 없이 매일 보는 습관형 구조
- 커뮤니티형 정보 큐레이션의 단순함

### 버릴 것

- 너무 개발자 커뮤니티 같은 투박함
- 부동산 초보자에게 차갑게 느껴질 수 있는 텍스트 과밀
- 복제 수준의 Hacker News 스타일

### 집집 적용

- `오늘의 집픽`을 GeekNews의 1번 글처럼 가장 먼저 읽게 만든다.
- 링크 카드는 뉴스 카드가 아니라 "집어볼 이슈"처럼 보이게 한다.
- 점수/댓글/출처 정보는 작게, 제목과 한 줄 해석은 크게 둔다.

## Reference 2. Axios

URL: https://www.axios.com/

### 가져올 것

- 짧은 문단 중심의 briefing 문법
- "Why it matters"형 설명 구조
- 빠르게 훑어도 요점이 남는 편집 방식
- 뉴스보다 해석이 앞에 오는 느낌

### 버릴 것

- 미국 정치/일반 뉴스 포털의 넓은 범위
- 큰 이미지 중심 기사 카드
- 브랜드 컬러/레이아웃 직접 복제

### 집집 적용

- 링크 상세와 홈 카드에 `왜 중요함`, `내 상황 영향`, `체크포인트`를 명확히 둔다.
- 긴 요약보다 짧은 브리핑 문장을 우선한다.
- "읽기 전부터 결론이 보이는" UI를 만든다.

## Reference 3. Bloomberg Markets

URL: https://www.bloomberg.com/markets

### 가져올 것

- 시장 상태를 숫자/상태/섹션으로 빠르게 보여주는 정보 위계
- 금융 정보 서비스 특유의 신뢰감
- 상단에서 오늘의 분위기를 먼저 읽게 하는 구조

### 버릴 것

- 기관 투자자용 과밀 화면
- 복잡한 차트와 티커 남발
- 어두운 터미널 감성 과잉

### 집집 적용

- 상단에 `오늘의 판세`, `이슈 강도`, `지역 흐름`을 배치한다.
- 숫자는 보조 정보로 쓰고, 개인 사용자가 이해할 문장 해석을 앞에 둔다.
- 금융 서비스의 단단함은 가져오되 무섭고 복잡한 화면은 피한다.

## Reference 4. XAI Land RADAR

URL: https://xai.land/products/radar/

### 가져올 것

- 조기 신호/리스크 감지라는 문제의식
- 지역별 위험/변화 신호를 보여주는 구조
- 부동산 시장을 "판세"로 읽는 관점

### 버릴 것

- `RADAR`, `레이더`, `조기경보 시스템` 같은 직접적 네이밍
- 기관/B2B 세일즈 페이지 느낌
- 예측 정확도나 리스크 모델을 가진 것처럼 보이는 표현

### 집집 적용

- "기관용 리스크 감시"가 아니라 "개인용 부동산 이슈 브리핑"으로 번역한다.
- `레이더 점수` 대신 `오늘의 중요도`, `이슈 강도`, `오늘의 집픽`을 쓴다.
- XAI Land와 겹치지 않게 브랜드와 문구는 `집집` 중심으로 간다.

## Final Direction

집집의 홈 화면은 아래 조합이어야 한다.

```text
GeekNews의 피드 밀도
+ Axios의 짧은 해석 문법
+ Bloomberg Markets의 시장 분위기 위계
+ XAI RADAR의 신호/리스크 관점
- 레이더/RADAR 직접 네이밍
- 회색 SaaS 카드밭
- 기관용 B2B 세일즈 톤
```

## Must Not Build

- 큰 hero + 카드 3개짜리 랜딩 페이지
- 회색 테두리 카드가 반복되는 무미건조한 SaaS 보드
- `DB 연결`, `MVP`, `실제 저장 지원` 같은 개발자 상태 화면
- `부동산 레이더`, `RADAR`, `레이더 점수`를 공개 홈의 핵심 표현으로 쓰는 화면
- 원문 제목만 쌓아둔 뉴스 목록
- "예쁘지만 오늘 뭘 봐야 하는지 모르는" 화면

## Must Build

- `집집` 브랜드가 첫 화면에 선명한 홈
- `오늘의 집픽`이 바로 보이는 구조
- 무주택자/1주택자/전세 세입자/투자자별 영향 요약
- 지역별 흐름
- 강한 신호 / 지켜볼 신호 / 소음 가능성
- 긴 뉴스보다 짧은 해석이 먼저 보이는 UI

# Impact Reference Addendum

현재 홈 리디자인의 실패 원인:

- 섹션 구성은 맞았지만 첫 화면에 감정적/인지적 한 방이 약했다.
- `집집` 브랜드가 크게 보이는 대신, 사용자가 오늘 무엇을 봐야 하는지 바로 꽂히지 않았다.
- `오늘의 집픽`이 있어도 화면의 주인공처럼 느껴지지 않았다.
- 숫자와 카드가 있지만 "오늘 판세가 뭐냐"는 결론이 약했다.

따라서 다음 루프는 기존 레퍼런스 4개에 아래 레퍼런스를 추가한다.

## Impact Reference 1. Dia Morning Brief

URL: https://www.diabrowser.com/

### 가져올 것

- "Start your day two steps ahead" 식의 하루 시작 브리핑 감각
- 앱의 기능 설명보다 "오늘 들어오면 무엇을 먼저 알게 되는가"를 전면에 두는 방식
- Morning Brief처럼 여러 정보를 모아 하나의 시작점으로 만들어주는 구조

### 집집 적용

- 홈 첫 문장은 `집집`이 아니라 오늘 판세 선언이어야 한다.
- 예: `오늘은 대출과 청약이 시장을 밀고 있습니다`
- 브랜드명은 상단/헤더에 작게 두고, 첫 화면의 주인공은 `오늘 결론`으로 둔다.

## Impact Reference 2. Robinhood / Toss Securities Style

URLs:
- https://robinhood.com/us/en/
- https://corp.tossinvest.com/en

### 가져올 것

- 금융 앱 특유의 큰 숫자, 강한 색, 즉시 읽히는 상태 변화
- 어려운 시장 정보를 일반 사용자가 "지금 올라감/내림/주의/기회"로 바로 읽게 하는 방식
- 투자 화면처럼 복잡하게 만들지 않고, 한 화면에서 심리적 방향을 잡아주는 구조

### 집집 적용

- `오늘의 중요도 92`만 놓지 말고, 왜 92인지 한 줄 verdict와 붙인다.
- 숫자는 보조가 아니라 감정적 시그널이어야 한다.
- 예:
  - `92`
  - `강한 신호`
  - `대출 규제 기대가 매수 심리를 자극 중`
- 상승/주의/관망 등 상태 색을 더 과감하게 써도 된다.

## Impact Reference 3. Redfin Data Center

URL: https://www.redfin.com/news/data-center/

### 가져올 것

- 부동산 데이터를 `Housing Market Tracker`, `Balance of Power`, `Price Drops`처럼 명확한 문제 단위로 나누는 방식
- 데이터 자체보다 사용자가 찾는 질문을 먼저 보여주는 구조
- 시장을 `가격`, `수요`, `협상력`, `취소`, `재등록` 같은 신호 묶음으로 읽는 방식

### 집집 적용

- 지역/카테고리 필터보다 먼저 "오늘의 신호 묶음"을 보여준다.
- 예:
  - `매수심리`
  - `전세압력`
  - `청약쏠림`
  - `지역온도`
- 피드는 뉴스 목록이 아니라 "신호 묶음별 이슈"여야 한다.

## Impact Reference 4. Zillow Market Heat Index

URL: https://www.zillow.com/research/market-heat-index-34054/

### 가져올 것

- 시장 온도를 하나의 지수/상태로 표현하는 방식
- 지역별 경쟁 강도, 수요/공급 균형을 "heat"로 읽게 만드는 방식
- 복잡한 데이터를 `뜨거움/식음/균형`처럼 빠르게 이해시키는 언어

### 집집 적용

- `지역 흐름`은 리스트 카드보다 `지역 온도판`에 가까워야 한다.
- 예:
  - `서울 HOT`
  - `경기/인천 WATCH`
  - `지방 중소도시 COOL`
- 텍스트 리스트보다 온도/상태 색/막대/랭킹을 써서 한눈에 들어오게 한다.

## Impact Reference 5. Realtor.com Hottest Markets

URL: https://www.realtor.com/research/topics/hottest-markets/

### 가져올 것

- "Hottest Markets"처럼 시장을 랭킹/승자/변화로 보여주는 언어
- 지역 이름 + 순위 변화 + 왜 뜨거운지 한 줄 설명하는 구조
- 매달/매주 다시 보게 만드는 반복형 포맷

### 집집 적용

- `지역 흐름`은 지역 카드 4개보다 `오늘 뜨는 지역 TOP` 느낌이 더 강해야 한다.
- 예:
  - `오늘 뜨는 지역 1위: 서울`
  - `정책/청약 신호 집중`
  - `5개 이슈 중 3개가 매수심리 관련`

## Impact Reference 6. Morning Brew / Axios Smart Brevity

URLs:
- https://www.morningbrew.com/
- https://www.axioshq.com/smart-brevity

### 가져올 것

- 매일 보는 짧은 브리프의 리듬
- 첫 줄에서 요점을 박고, 다음 줄에서 왜 중요한지 설명하는 문법
- 정보 과잉보다 "오늘 이거만 기억"을 만드는 편집

### 집집 적용

- 첫 화면에 `오늘 한 줄 결론`을 만든다.
- 모든 섹션은 아래 문법을 따른다:
  - `무슨 일`
  - `왜 중요`
  - `내 상황 영향`
- 카드 제목보다 결론 문장을 먼저 보여준다.

# Next Home Direction

다음 홈 루프는 아래 구조로 간다.

## 1. First View Must Hit

첫 화면에서 가장 큰 텍스트는 브랜드명이 아니라 `오늘 판세 결론`이다.

예:

```text
오늘은 대출과 청약이 시장을 밀고 있습니다
```

보조:

```text
매수심리 반등 신호 · 전세 세입자는 관망 · 서울/경기 쏠림
```

## 2. Hero Is Not Marketing, It Is Verdict

큰 hero 금지는 유지한다. 하지만 "큰 결론 블록"은 필요하다.

구조:

- 왼쪽: 오늘 판세 결론
- 오른쪽: 오늘의 중요도 / 강한 신호 수 / 위험 신호 수
- 아래: 무주택자/1주택자/전세 세입자/투자자 상황 탭

## 3. Today Pick Must Feel Like Breaking Market Signal

`오늘의 집픽`은 카드 하나가 아니라 속보/시장 시그널처럼 보여야 한다.

필수 요소:

- 큰 상태 라벨: `강한 신호`
- 큰 제목
- 한 줄 verdict
- `왜 중요함`은 1~2줄
- `내 상황 영향`은 선택된 상황에 맞춰 강하게 표시

## 4. Region Flow Must Become Heat Board

`지역 흐름`은 단순 카드 목록이 아니라 지역 온도판이어야 한다.

구조:

- 서울 `HOT`
- 경기/인천 `WATCH`
- 지방 광역시 `MIXED`
- 지방 중소도시 `COOL`

각 지역은 `막대`, `상태색`, `이슈 수`, `핵심 토픽`을 포함한다.

## 5. Feed Must Become Signal Tape

기존 3열 카드 피드는 PC에서는 괜찮지만 모바일에서 임팩트가 약하다.

다음 루프:

- 모바일: 한 줄씩 쌓이는 `Signal Tape`
- PC: 좌측 메인 브리프 + 우측 지역 온도판 + 하단 신호 테이프
- 카드 테두리 반복보다 `번호`, `중요도`, `상태`, `한 줄 해석` 위주

## Do Not Repeat

- 브랜드명만 크게 넣고 제품이 강해졌다고 착각하지 않는다.
- 흰 카드/회색 카드 반복으로 정리된 척하지 않는다.
- `오늘의 집픽`을 화면 중간 아래로 밀지 않는다.
- 숫자를 놓기만 하지 말고, 숫자의 의미를 바로 붙인다.
- "예쁜 정보판"이 아니라 "오늘 뭘 봐야 하는지 때리는 첫 화면"을 만든다.
